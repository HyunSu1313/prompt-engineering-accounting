# 01. 레퍼런스 검색용 프롬프트

회계처리 근거, 기준서 해석, 내부통제 중요성, ERP·엑셀 문제해결, 논리·계산 검증까지 — **레퍼런스(근거)에 기반한 모든 질의**를 위한 통합 프롬프트 템플릿입니다.

> **📌 준용 기준**
> 본 템플릿의 모든 답변은 아래 기준을 우선 준용합니다.
> - 한국채택국제회계기준(K-IFRS) 또는 일반기업회계기준(K-GAAP)
> - 내부회계관리제도 설계·운영 개념체계 (금융감독원)
> - 내부회계관리제도 평가 및 보고 가이드라인 (금감원)
> - 관련 감사기준서(KSA) 및 COSO 2013 프레임워크 (보완 참조)

---

## 📌 사용 목적

- 회계처리 방식의 근거를 기준서 수준에서 이해
- 특정 통제가 왜 중요한지, 미비점 수준은 어떻게 판단하는지 확인
- 엑셀 함수 오류, ERP 처리 방식 등 기술적 문제 해결
- 계산·해석·논리 흐름이 맞는지 검증
- 위 모든 결과를 검토멘트·감사조서에 바로 쓸 수 있는 문구로 출력

---

## 📝 기본 템플릿

```
다음 [질문 유형]에 대해 아래 형식으로 답변해줘.

[질문 유형]
{회계처리 / 내부통제 / ERP·엑셀 / 논리·계산 검증}

[회사 컨텍스트] (필요시 기재, 답변 기준이 달라지는 경우에만 적용)
- 상장 여부: {코스피 / 코스닥 / 비상장}
- 규모: {대형 / 중소기업}
- 업종: {업종}
- 회계처리기준: {K-IFRS / 일반기업회계기준(K-GAAP)}

[질문 또는 검증 내용]
{회계처리 내용 / 통제 내용 / 오류·문제 상황 / 검증받을 계산식·해석·논리}

[출력 형식]
1. 핵심 답변 (근거·원인·결과 등 질문 유형에 맞게)
2. 관련 근거
   - 기준서명 문단 번호
     원문: "해당 문단 원문 문장"
   - 질의회신:
     ① 공식 질의회신: 한국회계기준원 공식 사례번호 확인된 경우 번호와 요지 제시,
                      확인되지 않는 경우 "공식 질의회신 없음" 명시
     ② 참고 자료: 웹 검색으로 찾은 관련 자료가 있는 경우
                  "공식 질의회신은 아니나 참고 가능:" 으로 별도 표기 후 출처 명시
3. 실무 포인트 / 추가 고려사항
4. 미비점 수준 판단 (내부통제 관련 질문인 경우)
   - 단순 미비점 / 유의한 미비점 / 중요한 취약점 중 해당 수준 및 근거
5. 감사조서·검토멘트에 바로 쓸 수 있는 한 줄 문구

전문 용어는 유지하되, 실무자가 바로 활용할 수 있는 수준으로 작성해줘.
확인되지 않는 내용은 추정하지 말고 "직접 확인 필요"로 명시해줘.
```

## ✅ 사용 예시 1 — 회계처리 (확정기여형 퇴직급여)

**입력:**
```
[질문 유형] 회계처리

[질문]
확정기여형(DC) 퇴직급여는 지급 시점에 비용 처리하는 건지,
아니면 퇴직급여충당부채와 상계하는 건지 헷갈려.
확정급여형(DB)과 비교해서 분개까지 보여줘.

[출력 형식]
1. 회계처리 근거 (적용 기준서 포함)
2. 분개 예시 (차변/대변)
3. 실무 포인트 (놓치기 쉬운 부분)
4. 감사조서 또는 검토멘트에 쓸 수 있는 한 줄 문구
```

**출력 예시:**

1. **회계처리 근거**
   - 확정기여형: K-IFRS 제1019호 **문단44** — 기여금을 납부할 때 비용으로 인식. 추가적인 법적·의제 의무 없음. 충당부채 설정 불필요.
   - 확정급여형: K-IFRS 제1019호 **문단57** — 보험수리적 가정으로 확정급여채무(DBO)를 측정하고 순확정급여부채(자산) 인식.
   - 관련 질의회신: 공식 질의회신 없음. (참고 가능: "DC형 기여금은 근무용역 제공기간에 비용 인식" 취지의 세무 해석 자료 — 공식 한국회계기준원 사례번호 아님)

2. **분개 예시**

   확정기여형 (납부 시점):
   ```
   (차) 퇴직급여    xxx
   (대) 현금        xxx
   ```

   확정급여형 (기말 설정):
   ```
   (차) 퇴직급여    xxx
   (대) 확정급여부채 xxx
   ```

3. **실무 포인트**
   - DC형은 충당부채 잔액이 없어야 정상. 잔액 있으면 제도 전환 시 미상계 금액 여부 확인 필요. (K-IFRS 제1019호 문단44)
   - DB형은 보험수리사 보고서 수령 여부 및 할인율 적정성 검토 필요. (K-IFRS 제1019호 문단83)
   - DC형 전환 시 정산손익 처리 필요. (K-IFRS 제1019호 문단111)

4. **감사조서 문구**
   > "동사는 확정기여형 퇴직연금제도를 운영하고 있으며, 당기 기여금 납부액 XXX원을 퇴직급여로 비용 인식함. K-IFRS 1019호에 따라 추가적인 충당부채 설정 의무 없음을 확인함."

## ✅ 사용 예시 2 — 내부통제 (권한분리)

**입력:**
```
[질문 유형] 내부통제

[질문]
구매 요청, 승인, 지급을 동일인이 처리하면 안 되는 이유를
클라이언트에게 설명해야 해. RCM 문구랑 클라이언트 설명용 문구 둘 다 필요해.

[출력 형식]
1. 통제 목적
2. 대응 위험
3. 관련 근거
4. 미비점 수준 판단
5. RCM 작성용 한 줄 문구
6. 클라이언트 설명용 문구
```

**출력 예시:**

1. **통제 목적**
   구매 프로세스의 각 단계(요청·승인·지급)를 분리함으로써 단일 담당자가 전 과정을 단독 처리하여 발생할 수 있는 부정 및 오류를 예방

2. **대응 위험**
   - 허위 구매 요청 후 자기 승인·지급 → 횡령 위험
   - 오류 발생 시 상호 검증 불가 → 재무제표 왜곡 위험

3. **관련 근거**
   - 내부회계관리제도 설계·운영 개념체계 문단102: "기업은 부정 및 오류의 예방과 적발을 위하여 상충되는 직무를 서로 다른 개인에게 배분하여야 한다."
   - 내부회계관리제도 평가 및 보고 가이드라인 (금감원): "통제 미비점이 단독으로 또는 다른 미비점과 결합하여 재무제표의 중요한 왜곡표시를 초래할 가능성이 합리적으로 존재하는 경우 유의한 미비점으로 분류한다."
   - COSO 2013 원칙10 (보완 참조)
   - 질의회신: 공식 질의회신 없음

4. **미비점 수준 판단**
   - 구매 요청·승인·지급이 동일인에게 집중된 경우 → 유의한 미비점 이상 가능성 높음
   - 금액 규모, 보완통제 존재 여부에 따라 최종 수준 결정

5. **RCM 작성용 문구**
   > "구매 요청·승인·지급 담당자를 분리하여 단일 담당자의 단독 처리를 방지하고, 상호 검증을 통해 부정 및 오류 발생 위험을 통제함."

6. **클라이언트 설명용 문구**
   > "한 사람이 돈을 요청하고, 결재하고, 지급까지 혼자 처리하면 견제가 없어져요. 구매 요청·승인·지급을 각각 다른 사람이 하도록 역할을 나누는 게 권한분리 통제입니다."

## ✅ 사용 예시 3 — ERP·엑셀 (VLOOKUP 오류)

**입력:**
```
[질문 유형] ERP·엑셀

[질문]
VLOOKUP으로 거래처 코드를 매핑하려는데 값이 있는데도 #N/A 오류가 떠.
=VLOOKUP(A2, $D$2:$E$100, 2, 0) 이렇게 썼어.

[출력 형식]
1. 원인 분석
2. 해결 방법
3. 재발 방지 팁
4. 관련 함수 추천
```

**출력 예시:**

1. **원인 분석**
   #N/A 오류의 주요 원인: A2와 D열 값의 데이터 형식 불일치(숫자 vs 텍스트), 앞뒤 공백 포함

2. **해결 방법**
   ```excel
   =VLOOKUP(TRIM(TEXT(A2,"0")), $D$2:$E$100, 2, 0)
   ```

3. **재발 방지 팁**
   - ERP에서 데이터 추출 시 숫자 코드가 텍스트로 저장되는 경우 많음
   - 매핑 전 `=ISNUMBER(A2)`로 형식 확인 습관화

4. **관련 함수 추천**
   - `XLOOKUP`: VLOOKUP보다 유연하고 오류 처리 쉬움
   ```excel
   =XLOOKUP(TRIM(A2), $D$2:$D$100, $E$2:$E$100, "없음")
   ```

## ✅ 사용 예시 4 — 논리·계산 검증

**입력:**
```
[질문 유형] 논리·계산 검증

[검증 내용]
감가상각 계산이 맞는지 확인해줘.
- 취득원가 10,000,000원, 잔존가치 0원, 내용연수 5년, 정액법
- 취득일 2023년 7월 1일, 결산일 2023년 12월 31일
- 첫 해 감가상각비 = 10,000,000 ÷ 5 × (6/12) = 1,000,000원

[출력 형식]
1. 검증 결과 (맞음 / 틀림 / 부분적으로 맞음)
2. 근거
3. 추가 고려사항
```

**출력 예시:**

1. **검증 결과** ✅ 맞음
   - 10,000,000 ÷ 5 = 연간 2,000,000원 × 6/12(7~12월) = 1,000,000원

2. **근거**
   - K-IFRS 제1016호 문단55: "감가상각은 자산이 사용 가능한 때부터 시작"

3. **추가 고려사항**
   - K-IFRS는 월할 상각이 원칙 (일할 계산 회사도 있으니 회계정책 확인)
   - 세무상 감가상각과 회계상 감가상각 차이 → 이연법인세 검토 필요 (K-IFRS 제1012호 문단15)
