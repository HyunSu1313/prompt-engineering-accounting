# 05. 논리 · 계산 검증 프롬프트

계산 검증, 해석 검증, 표현 검증 등 내가 이해한 논리가 맞는지 확인하기 위한 프롬프트 템플릿입니다.

> **📌 준용 기준**
> 본 템플릿의 모든 답변은 아래 기준을 우선 준용합니다.
> - 한국채택국제회계기준(K-IFRS) 또는 일반기업회계기준(K-GAAP)
> - 내부회계관리제도 설계·운영 개념체계 (금융감독원)
> - 내부회계관리제도 평가 및 보고 가이드라인 (금감원)
> - 관련 감사기준서(KSA) 및 COSO 2013 프레임워크 (보완 참조)

---

## 📌 사용 목적

- 내가 한 계산이 맞는지 검증
- 회계·감사 개념 이해가 정확한지 확인
- 보고서·조서의 논리 흐름이 맞는지 점검

---

## 📎 정확한 원문 인용을 위한 PDF 첨부 안내

관련 근거의 원문을 정확하게 인용하려면 아래 기준서 PDF를 프롬프트에 첨부하세요.
Claude가 PDF 내용을 직접 읽고 해당 문단 원문을 그대로 인용합니다.

| 기준 | 출처 |
|---|---|
| K-IFRS 각 기준서 | kasb.or.kr |
| 내부회계관리제도 설계·운영 개념체계 | k-icfr.org |
| 내부회계관리제도 평가 및 보고 가이드라인 (금감원) | fss.or.kr |
| 감사기준서(KSA) | kicpa.or.kr |

> ⚠️ PDF 미첨부 시 Claude가 원문을 추정하여 출력할 수 있습니다.
> 이 경우 반드시 실제 기준서와 대조·검증 후 사용하세요.

## 📝 기본 템플릿

```
다음 내용이 맞는지 검증해줘.

[검증 유형]
{계산 검증 / 해석 검증 / 표현 검증 / 논리 검증}

[내용]
{검증받을 계산식, 해석, 표현, 또는 논리}

[출력 형식]
1. 검증 결과 (맞음 / 틀림 / 부분적으로 맞음)
2. 틀린 부분 또는 보완이 필요한 부분
3. 올바른 내용 또는 개선 방향
4. 추가로 고려해야 할 사항
5. 관련 근거
   - 기준서명 문단 번호
     원문: "해당 문단 원문 문장"
     
   - 질의회신:
     ① 공식 질의회신: 한국회계기준원 공식 사례번호 확인된 경우 번호와 요지 제시,
                      확인되지 않는 경우 "공식 질의회신 없음" 명시
     ② 참고 자료: 웹 검색으로 찾은 관련 자료가 있는 경우
                  "공식 질의회신은 아니나 참고 가능:" 으로 별도 표기 후 출처 명시

틀린 경우 왜 틀렸는지 논리적으로 설명해줘.
```

## ✅ 사용 예시 1 — 계산 검증

**입력:**
```
[검증 유형] 계산 검증

[내용]
감가상각 계산이 맞는지 확인해줘.
- 취득원가: 10,000,000원
- 잔존가치: 0원
- 내용연수: 5년
- 상각방법: 정액법
- 취득일: 2023년 7월 1일
- 결산일: 2023년 12월 31일

첫 해 감가상각비 = 10,000,000 ÷ 5 × (6/12) = 1,000,000원

[출력 형식]
1. 검증 결과
2. 틀린 부분
3. 올바른 내용
4. 추가 고려사항
```

**출력 예시:**

1. **검증 결과** ✅ 맞음

2. **확인 사항**
   - 취득원가 10,000,000 ÷ 내용연수 5년 = 연간 2,000,000원
   - 7월 취득 → 당기 사용월수 6개월 (7~12월)
   - 2,000,000 × 6/12 = **1,000,000원** ✅

3. **추가 고려사항**
   - K-IFRS는 월할 상각이 원칙 (일할 계산 회사도 있으니 회계정책 확인)
     → K-IFRS 제1016호 문단55: "감가상각은 자산이 사용 가능한 때부터 시작"
       링크: https://kasb.or.kr (기준서 → K-IFRS 제1016호)
   - 잔존가치 0원 정책인지 주석 확인 필요
     → K-IFRS 제1016호 문단73(c): 유형자산 분류별 잔존가치 공시 요구
       링크: https://kasb.or.kr (기준서 → K-IFRS 제1016호)
   - 세무상 감가상각과 회계상 감가상각 차이 → 이연법인세 검토 필요
     → K-IFRS 제1012호 문단15: 일시적차이에 대한 이연법인세부채 인식
       링크: https://kasb.or.kr (기준서 → K-IFRS 제1012호)

## ✅ 사용 예시 2 — 해석 검증

**입력:**
```
[검증 유형] 해석 검증

[내용]
내가 이해한 게 맞는지 확인해줘.

"중요성 금액 이하의 오류는 재무제표에 영향을 미치지 않으므로
수정 없이 넘어가도 된다."

[출력 형식]
1. 검증 결과
2. 틀린 부분 또는 보완이 필요한 부분
3. 올바른 내용
4. 추가 고려사항
```

**출력 예시:**

1. **검증 결과** ⚠️ 부분적으로 맞음

2. **보완이 필요한 부분**
   중요성 이하라고 해서 무조건 수정 없이 넘어가는 건 아님. 집합적 영향(aggregation) 고려 필요.

3. **올바른 내용**
   - 개별 오류가 중요성 이하이더라도 **여러 오류의 합산이 중요성을 초과하면** 수정 필요
   - 또한 질적 중요성(부정 관련, 경영진 보수 관련 등)은 금액과 무관하게 중요할 수 있음
   - 감사인은 수정되지 않은 왜곡표시 누계액을 집계하여 최종 판단

4. **추가 고려사항**
   - 집합적 왜곡표시 평가 근거:
     → KSA 450 문단5: 감사인은 수정되지 않은 왜곡표시의 누계액이 중요한지 평가
       링크: https://kicpa.or.kr (감사기준서 → KSA 450)
   - 질적 중요성 근거:
     → KSA 450 문단11: 금액과 무관하게 성격상 중요할 수 있는 왜곡표시 고려
       링크: https://kicpa.or.kr (감사기준서 → KSA 450)
   - 조서에 "중요성 이하로 수정 불요"라고만 기재하면 집합적 검토 누락 가능성 있음
