# 01. 회계처리 · 기준서 해석 프롬프트

회계처리 근거, 기준서 해석, 분개 논리를 실무에서 바로 쓸 수 있는 형태로 설명받기 위한 프롬프트 템플릿입니다.

> **📌 준용 기준**
> 본 템플릿의 모든 답변은 아래 기준을 우선 준용합니다.
> - 한국채택국제회계기준(K-IFRS) 또는 일반기업회계기준(K-GAAP)
> - 내부회계관리제도 설계·운영 개념체계 (금융감독원)
> - 내부회계관리제도 평가 및 보고 가이드라인 (금감원)
> - 관련 감사기준서(KSA) 및 COSO 2013 프레임워크 (보완 참조)

---

## 📌 사용 목적

- 회계처리 방식의 근거를 기준서 수준에서 이해
- 검토멘트·감사조서에 바로 쓸 수 있는 문구로 출력
- 유사 케이스 비교 설명 (예: 확정기여형 vs 확정급여형)

---

## 📎 정확한 원문 인용을 위한 PDF 첨부 안내

관련 근거의 원문을 정확하게 인용하려면 아래 기준서 PDF를 프롬프트에 첨부하세요.
Claude가 PDF 내용을 직접 읽고 해당 문단 원문을 그대로 인용합니다.

| 기준 | 출처 |
|---|---|
| K-IFRS 각 기준서 | kasb.or.kr |
| 내부회계관리제도 설계·운영 개념체계 | k-icfr.org |
| 내부회계관리제도 평가 및 보고 가이드라인 (금감원) | fss.or.kr |
| 감사기준서(KSA) | kicpa.or.kr |

> ⚠️ PDF 미첨부 시 Claude가 원문을 추정하여 출력할 수 있습니다.
> 이 경우 반드시 실제 기준서와 대조·검증 후 사용하세요.

## 📝 기본 템플릿

```
다음 회계처리에 대해 아래 형식으로 설명해줘.

[질문]
{회계처리 내용 또는 궁금한 사항}

[출력 형식]
1. 회계처리 근거
   - 기준서명 문단 번호
     원문: "해당 문단 원문 문장"

   - 관련 질의회신:
     ① 공식 질의회신: 한국회계기준원 공식 사례번호가 확인되는 경우 번호와 요지 제시,
                      확인되지 않는 경우 "공식 질의회신 없음" 명시
     ② 참고 자료: 웹 검색을 통해 찾은 관련 자료가 있는 경우
                  "공식 질의회신은 아니나 참고 가능:" 으로 별도 표기 후 출처 명시
2. 분개 예시 (차변/대변)
3. 실무 포인트 (놓치기 쉬운 부분)
4. 감사조서 또는 검토멘트에 쓸 수 있는 한 줄 문구

전문 용어는 유지하되, 실무자가 바로 활용할 수 있는 수준으로 작성해줘.
```

## ✅ 사용 예시 1 — 확정기여형 퇴직급여

**입력:**
```
다음 회계처리에 대해 아래 형식으로 설명해줘.

[질문]
확정기여형(DC) 퇴직급여는 지급 시점에 비용 처리하는 건지,
아니면 퇴직급여충당부채와 상계하는 건지 헷갈려.
확정급여형(DB)과 비교해서 분개까지 보여줘.

[출력 형식]
1. 회계처리 근거 (적용 기준서 포함)
2. 분개 예시 (차변/대변)
3. 실무 포인트 (놓치기 쉬운 부분)
4. 감사조서 또는 검토멘트에 쓸 수 있는 한 줄 문구
```

**출력 예시 (Claude 응답):**

1. **회계처리 근거**
   - 확정기여형: K-IFRS 제1019호 **문단44** — 기여금을 납부할 때 비용으로 인식. 추가적인 법적·의제 의무 없음. 충당부채 설정 불필요.

   - 확정급여형: K-IFRS 제1019호 **문단57** — 보험수리적 가정으로 확정급여채무(DBO)를 측정하고 순확정급여부채(자산) 인식.

   - 관련 질의회신:
     ① 공식 질의회신: 공식 질의회신 없음
     ② 참고 자료: 공식 질의회신은 아니나 참고 가능 —
        "DC형 기여금은 근무용역 제공기간에 비용 인식" 취지의 질의회신 사례 존재
        (출처: 국세청 세무 해석 자료 / taxnet.co.kr 수록 — 공식 한국회계기준원 사례번호 아님)

2. **분개 예시**

   확정기여형 (납부 시점):
   ```
   (차) 퇴직급여    xxx
   (대) 현금        xxx
   ```

   확정급여형 (기말 설정):
   ```
   (차) 퇴직급여    xxx
   (대) 확정급여부채 xxx
   ```

3. **실무 포인트**
   - DC형은 충당부채 잔액이 없어야 정상. 잔액 있으면 제도 전환 시 미상계 금액 여부 확인 필요.
     → K-IFRS 제1019호 문단44: DC형은 납부 기여금 외 추가 의무 없음
   - DB형은 보험수리사 보고서 수령 여부 및 할인율 적정성 검토 필요.
     → K-IFRS 제1019호 문단83: 우량회사채 시장수익률 기준 할인율 결정
  
   - DC형 전환 시 정산손익 처리 필요
     → K-IFRS 제1019호 문단111: 제도 정산 시 손익 인식 요건
  

4. **감사조서 문구**
   > "동사는 확정기여형 퇴직연금제도를 운영하고 있으며, 당기 기여금 납부액 XXX원을 퇴직급여로 비용 인식함. K-IFRS 1019호에 따라 추가적인 충당부채 설정 의무 없음을 확인함."

## ✅ 사용 예시 2 — 임대보증금 현재가치 평가

**입력:**
```
다음 회계처리에 대해 아래 형식으로 설명해줘.

[질문]
임대보증금을 현재가치로 평가해야 하는 경우와
그렇지 않은 경우의 기준이 뭐야?
중요성 판단 기준도 포함해줘.

[출력 형식]
1. 회계처리 근거 (적용 기준서 포함)
2. 분개 예시 (차변/대변)
3. 실무 포인트 (놓치기 쉬운 부분)
4. 감사조서 또는 검토멘트에 쓸 수 있는 한 줄 문구
```

## 🔧 변형 템플릿 — 빠른 한 줄 질문용

간단한 확인이 필요할 때 사용합니다.

```
[회계처리 확인]
{처리 내용}

위 처리가 K-IFRS / 일반기업회계기준 상 적정한지 확인해줘.
틀렸다면 올바른 처리 방법과 근거 기준서를 알려줘.
감사조서에 쓸 수 있는 한 줄 문구도 포함해줘.
```
