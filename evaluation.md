# Prompt Evaluation

이 라이브러리의 프롬프트는 "그냥 질문"과 비교해 실제로 더 나은 결과를 내는지를
아래 5가지 기준으로 평가합니다. 일반 질문 방식과 템플릿 적용 방식을 비교한
테스트 케이스를 통해 개선점을 검증합니다.

---

## 평가 기준

| # | 기준 | 설명 |
|---|---|---|
| 1 | 근거 정확성 | 기준서명·문단 번호가 실제 자료와 일치하는가 |
| 2 | 출력 일관성 | 요청한 출력 형식대로 답변하는가 |
| 3 | 실무 활용성 | 감사조서·검토멘트·메일에 바로 활용 가능한가 |
| 4 | 할루시네이션 통제 | 불확실한 내용에 대해 "직접 확인 필요"로 표시하는가 |
| 5 | 커뮤니케이션 품질 | 수신자와 상황에 맞는 문체·격식인가 |

---

## 테스트 케이스

| Case | 질문 유형 | 일반 질문 결과 | 템플릿 적용 결과 | 개선점 |
|---|---|---|---|---|
| 1 | DC형 퇴직급여 회계처리 (RCM 상계처리 적정성 검증) | 기준서 근거 불명확, 일반론적 답변 | K-IFRS 1019 문단 번호 기준으로 구조화, DC/DB 비교 분개 포함 | 근거 명시 + RCM 수정 문구 즉시 활용 가능 |
| 2 | 권한분리 통제 설명 | 통제 개념 설명은 가능하나 RCM 문구·미비점 수준 판단 누락 | 통제목적·대응위험·미비점수준·RCM문구·클라이언트설명 5단 분리 출력 | 실무 적용성 개선 |
| 3 | 클라이언트 자료 요청 메일 | 문장이 장황하고 격식 수준 불균일 | 요청자료·기한·수신자 톤 정리, 바로 발송 가능한 형태 | 커뮤니케이션 효율 개선 |
| 4 | VLOOKUP #N/A 오류 해결 | 가능한 원인 나열에 그침 | 원인분석→해결코드→재발방지팁→대안함수 순서로 구조화 | 즉시 적용 가능한 코드 제공 |
| 5 | 감가상각누계액 재계산 검증 (취득월 누락 발견) | 맞는지 틀린지만 답변, 누락 개월수·금액 특정 안 됨 | 누락 개월수·과소계상액 정량 특정 + 미비점 검토 필요성까지 제시 | 단순 계산 오류를 넘어 통제 관점 시사점까지 도출 |

## 평가 방법 (How)

1. 동일한 업무 상황에 대해 **일반 질문**과 **템플릿 적용 질문**을 각각 입력
2. 위 5가지 기준에 따라 출력 결과를 비교
3. 템플릿이 실제로 개선을 가져오지 못하는 경우, 템플릿 구조 자체를 수정
4. 위 절차를 거친 케이스만 README 및 카테고리 파일의 "사용 예시"에 등록

> 라이브러리에 실린 모든 "사용 예시"는 위 절차를 거쳐 비교·검토한 결과입니다.
> 검증 과정이 완료된 사례는 `case_studies/` 폴더에 상세 기록하며, 검증되지 않은 템플릿은 라이브러리 본문에 포함하지 않습니다.

## 한계 및 향후 계획

- 현재 평가는 정성적 비교(사람이 직접 비교) 방식이며, 정량적 채점 체계는 아직 없음
- 현재 상세 케이스는 2건(`case_studies/`)으로 일반화에는 한계가 있으며, 실무 적용 과정에서 케이스를 지속적으로 누적할 예정
- 케이스 2(감가상각 재계산)처럼 오류의 정량적 특정은 가능하나 **근본 원인까지는 확정되지 않는 경우**, 결과를 과장하지 않고 "추가 확인 필요"로 명시하는 것을 원칙으로 함
