# Case 01. DC형 퇴직급여 회계처리 검토

## 1. 업무 상황

확정기여형(DC)과 확정급여형(DB) 퇴직연금제도를 **동시에 운영**하는 회사의 RCM 회계처리 부분을 검토하는 중, DC형 퇴직연금 납부액을 퇴직급여충당부채와 **상계 처리**한다고 기재되어 있는 것을 발견하여, 이 처리가 K-IFRS상 적정한지 직접 검증해본 상황.

---

## 2. 일반 질문 방식

**질문:**
```
DC형 퇴직급여 회계처리 알려줘.
```

**한계:**
- 기준서 문단 번호가 명확하지 않음
- DB형과의 차이가 실무적으로 구분되지 않음
- 감사조서 문구로 바로 쓰기 어려움

---

## 3. 템플릿 적용 방식

**입력:**
```
[질문 유형] 회계처리

[회사 컨텍스트]
- 회계처리기준: K-IFRS

[질문]
당사는 확정기여형(DC)과 확정급여형(DB)을 동시에 운영하고 있는데,
기존 RCM에는 "DC형 퇴직연금 납부액을 퇴직급여충당부채와 상계 처리한다"고
기재되어 있어. 이 처리가 K-IFRS상 적정한지 검토해줘.
DB형과 비교해서 분개까지 보여줘.

[출력 형식]
1. 회계처리 근거
2. 분개 예시 (차변/대변)
3. 실무 포인트
4. RCM 수정 문구
```

---

## 4. 출력 결과

1. **회계처리 근거**
   - 확정기여형: K-IFRS 제1019호 문단44 — 기여금을 납부할 때 비용으로 인식. 추가적인 법적·의제 의무 없음. 충당부채 설정 불필요.
   - 확정급여형: K-IFRS 제1019호 문단57 — 보험수리적 가정으로 확정급여채무(DBO)를 측정.

2. **분개 예시**

   확정기여형 (납부 시점):
   ```
   (차) 퇴직급여    xxx
   (대) 현금        xxx
   ```

   확정급여형 (기말 설정):
   ```
   (차) 퇴직급여    xxx
   (대) 확정급여부채 xxx
   ```

3. **실무 포인트**
   - DC형은 충당부채 잔액이 없어야 정상. **DC·DB 동시 운영 회사에서는 DB형 충당부채 관리 로직이 RCM 작성 시 DC형 항목에도 그대로 옮겨 기재되는 오류가 발생하기 쉬움.**
   - 이번 케이스처럼 두 제도를 함께 운영하는 회사는, RCM 상 DC와 DB 처리 로직이 프로세스 단계에서 명확히 분리되어 기술되어 있는지 별도 확인 필요
   - → RCM 상 DC형 항목의 해당 문구는 "기여금 납부 시점에 비용으로 인식, 충당부채 상계 없음"으로 수정 필요 (DB형 항목과는 별도로 구분 기재)

4. **RCM 수정 문구**
   > "당사는 확정기여형 퇴직연금제도를 운영하고 있으며, 기여금 납부 시점에 퇴직급여로 비용 인식함. K-IFRS 제1019호에 따라 추가적인 충당부채 설정 의무가 없으므로 퇴직급여충당부채와의 상계 처리는 발생하지 않음."
