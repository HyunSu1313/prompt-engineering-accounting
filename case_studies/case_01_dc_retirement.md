# Case 01. DC형 퇴직급여 회계처리 검토

## 1. 업무 상황

클라이언트가 확정기여형(DC) 퇴직연금 납부액을 퇴직급여충당부채와 상계 처리해야 하는지 문의한 상황.

---

## 2. 일반 질문 방식

**질문:**
```
DC형 퇴직급여 회계처리 알려줘.
```

**한계:**
- 기준서 문단 번호가 명확하지 않음
- DB형과의 차이가 실무적으로 구분되지 않음
- 감사조서 문구로 바로 쓰기 어려움

---

## 3. 템플릿 적용 방식

**입력:**
```
[질문 유형] 회계처리

[회사 컨텍스트]
- 회계처리기준: K-IFRS

[질문]
확정기여형(DC) 퇴직급여는 지급 시점에 비용 처리하는 건지,
아니면 퇴직급여충당부채와 상계하는 건지 검토해줘.
확정급여형(DB)과 비교해서 분개까지 보여줘.

[출력 형식]
1. 회계처리 근거
2. 분개 예시 (차변/대변)
3. 실무 포인트
4. 감사조서 또는 검토멘트에 쓸 수 있는 한 줄 문구
```

---

## 4. 출력 결과

1. **회계처리 근거**
   - 확정기여형: K-IFRS 제1019호 문단44 — 기여금을 납부할 때 비용으로 인식. 추가적인 법적·의제 의무 없음. 충당부채 설정 불필요.
   - 확정급여형: K-IFRS 제1019호 문단57 — 보험수리적 가정으로 확정급여채무(DBO)를 측정.

2. **분개 예시**

   확정기여형 (납부 시점):
   ```
   (차) 퇴직급여    xxx
   (대) 현금        xxx
   ```

   확정급여형 (기말 설정):
   ```
   (차) 퇴직급여    xxx
   (대) 확정급여부채 xxx
   ```

3. **실무 포인트**
   - DC형은 충당부채 잔액이 없어야 정상. 잔액 있으면 제도 전환 시 미상계 금액 여부 확인 필요.
   - DB형 전환 이력이 있는 회사는 과거 DB형 충당부채 잔액 처리 방식을 별도 확인해야 함.

4. **감사조서 문구**
   > "동사는 확정기여형 퇴직연금제도를 운영하고 있으며, 당기 기여금 납부액 XXX원을 퇴직급여로 비용 인식함. K-IFRS 1019호에 따라 추가적인 충당부채 설정 의무 없음을 확인함."

---

## 5. 검토 결과

| 평가 기준 | 일반 질문 | 템플릿 적용 |
|---|---|---|
| 근거 정확성 | 기준서 문단 번호 없음 | K-IFRS 1019호 문단44, 57 명시 |
| 출력 일관성 | 형식 비고정 | 근거→분개→포인트→문구 순서 고정 |
| 실무 활용성 | 추가 가공 필요 | 조서 문구 그대로 사용 가능 |

템플릿 적용 결과가 근거 정확성, 출력 일관성, 실무 활용성 측면에서 일반 질문보다 우수함.
