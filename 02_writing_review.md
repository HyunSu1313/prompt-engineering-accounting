# 02. 문장 검토 · 실무 표현 프롬프트

검토멘트, 감사조서 문구, 사내 메일, 회계팀 문의 문장 등이 실무적으로 자연스러운지 검토하고 개선하기 위한 프롬프트 템플릿입니다.

> **📌 준용 기준**
> 본 템플릿의 모든 답변은 아래 기준을 우선 준용합니다.
> - 한국채택국제회계기준(K-IFRS) 또는 일반기업회계기준(K-GAAP)
> - 내부회계관리제도 설계·운영 개념체계 (금융감독원)
> - 내부회계관리제도 평가 및 보고 가이드라인 (금감원)
> - 관련 감사기준서(KSA) 및 COSO 2013 프레임워크 (보완 참조)

---

## 📌 사용 목적

- 작성한 문장이 실무 맥락에서 어색하지 않은지 확인
- 격식체·전문용어 수준 조정
- 검토멘트·조서 문구·메일 문장을 바로 쓸 수 있는 형태로 다듬기

---

## 📎 정확한 원문 인용을 위한 PDF 첨부 안내

회계처리 또는 주석공시 관련 표현이 포함된 경우, 관련 기준서 PDF를 첨부하면
Claude가 해당 문단 원문을 정확하게 인용합니다.

| 기준 | 출처 |
|---|---|
| K-IFRS 각 기준서 | kasb.or.kr |
| 내부회계관리제도 설계·운영 개념체계 | k-icfr.org |
| 내부회계관리제도 평가 및 보고 가이드라인 (금감원) | fss.or.kr |
| 감사기준서(KSA) | kicpa.or.kr |

> ⚠️ PDF 미첨부 시 Claude가 원문을 추정하여 출력할 수 있습니다.
> 이 경우 반드시 실제 기준서와 대조·검증 후 사용하세요.

---

## 📝 기본 템플릿

```
다음 문장을 실무 관점에서 검토해줘.

[문장 유형]
{검토멘트 / 감사조서 문구 / 사내 메일 / 회계팀 문의 / 기타}

[원문]
{검토받을 문장}

[출력 형식]
1. 어색한 부분 및 이유
2. 개선 문장 (바로 사용 가능한 형태)
3. 추가 고려사항 (수신자, 격식 수준, 문서 성격에 따른 조언)
   - 회계처리 또는 주석공시 관련 표현이 포함된 경우 관련 근거 반드시 아래 형식으로 첨부:
     기준서명 문단 번호
     원문: "해당 문단 원문 문장"

   - 질의회신:
     ① 공식 질의회신: 한국회계기준원 공식 사례번호 확인된 경우 번호와 요지 제시,
                      확인되지 않는 경우 "공식 질의회신 없음" 명시
     ② 참고 자료: 웹 검색으로 찾은 관련 자료가 있는 경우
                  "공식 질의회신은 아니나 참고 가능:" 으로 별도 표기 후 출처 명시

전문용어는 유지하되, 실무자가 바로 활용할 수 있는 수준으로 작성해줘.
```

---

## ✅ 사용 예시 1 — 검토멘트

**입력:**
```
[문장 유형] 검토멘트

[원문]
당사는 확정기여형 퇴직연금제도를 운용하고 있는바, 당기 납부한 기여금을
퇴직급여 비용으로 인식하였으며 이는 적정한 것으로 판단됩니다.

[출력 형식]
1. 어색한 부분 및 이유
2. 개선 문장
3. 추가 고려사항
```

**출력 예시:**

1. **어색한 부분**
   - "운용하고 있는바" → 구어체와 문어체가 혼재. 조서 문구에서는 "운영하고 있으며"가 자연스러움
   - "적정한 것으로 판단됩니다" → 수동적 표현. "적정함을 확인함"이 조서 어조에 맞음

2. **개선 문장**
   > "동사는 확정기여형 퇴직연금제도를 운영하고 있으며, 당기 납부 기여금 XXX원을 퇴직급여로 비용 인식함. K-IFRS 제1019호 문단44에 따른 처리로 적정함을 확인함."

3. **추가 고려사항**
   - 감사조서는 "~함", "~확인함" 종결어미 통일 권장
   - 금액 명기 가능 시 구체적 수치 포함할 것
   - "비용 인식" 표현의 근거:
     → K-IFRS 제1019호 문단44: 확정기여형 기여금 납부 시 비용 인식
       링크: https://kasb.or.kr (기준서 → K-IFRS 제1019호)
   - "충당부채 미계상 적정" 표현의 근거:
     → K-IFRS 제1019호 문단44: DC형은 납부 기여금 외 추가 의무 없음
       링크: https://kasb.or.kr (기준서 → K-IFRS 제1019호)

## ✅ 사용 예시 2 — 사내 메일

**입력:**
```
[문장 유형] 사내 메일 (회계팀 문의)

[원문]
안녕하세요. 이번에 퇴직급여 관련해서 여쭤볼 게 있어서요.
DC형인데 충당금이 남아있는 경우 어떻게 처리해야 하나요?

[출력 형식]
1. 어색한 부분 및 이유
2. 개선 문장
3. 추가 고려사항
```

---

## 🔧 변형 템플릿 — 문장 새로 작성 요청

문장을 검토가 아닌 새로 작성할 때 사용합니다.

```
다음 상황에 맞는 문장을 작성해줘.

[문장 유형]
{검토멘트 / 감사조서 문구 / 사내 메일 / 회계팀 문의}

[상황]
{전달할 내용 또는 맥락}

[조건]
- 격식 수준: {높음 / 보통 / 낮음}
- 수신자: {상급자 / 동료 / 외부 클라이언트}
- 분량: {한 줄 / 2~3줄 / 단락}

바로 사용 가능한 형태로 작성해줘.
```
