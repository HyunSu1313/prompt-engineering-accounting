# 04. ERP · 엑셀 문제해결 프롬프트

ERP 시스템 처리 방식, 엑셀 함수 오류, 단축키 등 실무에서 마주치는 기술적 문제를 빠르게 해결하기 위한 프롬프트 템플릿입니다.

> **📌 준용 기준**
> 본 템플릿의 모든 답변은 아래 기준을 우선 준용합니다.
> - 한국채택국제회계기준(K-IFRS) 또는 일반기업회계기준(K-GAAP) (회계처리 연관 시)
> - 내부회계관리제도 설계·운영 개념체계 (금융감독원) (내부통제 연관 시)
> - 내부회계관리제도 평가 및 보고 가이드라인 (금감원) (미비점 판단 연관 시)

---

## 📌 사용 목적

- 엑셀 함수 오류 원인 파악 및 해결
- ERP 처리 방식 이해 (왜 이렇게 작동하는지)
- 업무 효율화를 위한 함수·단축키 추천

---

## 📎 정확한 원문 인용을 위한 PDF 첨부 안내

ERP 처리 방식이 회계처리 또는 내부통제와 연관된 경우,
관련 기준서 PDF를 첨부하면 Claude가 해당 문단 원문을 정확하게 인용합니다.

| 기준 | 출처 |
|---|---|
| K-IFRS 각 기준서 | kasb.or.kr |
| 내부회계관리제도 설계·운영 개념체계 | k-icfr.org |
| 내부회계관리제도 평가 및 보고 가이드라인 (금감원) | fss.or.kr |

> ⚠️ PDF 미첨부 시 Claude가 원문을 추정하여 출력할 수 있습니다.
> 이 경우 반드시 실제 기준서와 대조·검증 후 사용하세요.

## 📝 기본 템플릿

```
다음 문제를 해결해줘.

[문제 유형]
{엑셀 함수 오류 / ERP 처리 방식 / 단축키 / 기타}

[상황]
{현재 상황 또는 오류 내용}

[출력 형식]
1. 원인 분석 (왜 이렇게 되는지)
2. 해결 방법 (단계별로)
3. 재발 방지 팁
4. 관련 함수·기능 추천 (더 효율적인 방법이 있으면)
5. 관련 근거 (ERP 처리 방식이 회계처리 또는 내부통제와 연관된 경우)
   - 기준서명 문단 번호
     원문: "해당 문단 원문 문장"
     
   - 질의회신:
     ① 공식 질의회신: 한국회계기준원 공식 사례번호 확인된 경우 번호와 요지 제시,
                      확인되지 않는 경우 "공식 질의회신 없음" 명시
     ② 참고 자료: 웹 검색으로 찾은 관련 자료가 있는 경우
                  "공식 질의회신은 아니나 참고 가능:" 으로 별도 표기 후 출처 명시

바로 적용할 수 있는 수준으로 설명해줘.
```

## ✅ 사용 예시 1 — 엑셀 VLOOKUP 오류

**입력:**
```
[문제 유형] 엑셀 함수 오류

[상황]
VLOOKUP으로 거래처 코드를 매핑하려는데 값이 있는데도 #N/A 오류가 떠.
=VLOOKUP(A2, $D$2:$E$100, 2, 0) 이렇게 썼어.

[출력 형식]
1. 원인 분석
2. 해결 방법
3. 재발 방지 팁
4. 관련 함수 추천
```

**출력 예시:**

1. **원인 분석**
   #N/A 오류의 주요 원인:
   - A2와 D열 값의 데이터 형식 불일치 (숫자 vs 텍스트)
   - 앞뒤 공백 포함 (눈에 안 보이지만 실제로 다른 값)
   - 대소문자 차이 (VLOOKUP은 대소문자 구분 없지만 특수문자 포함 시 영향)

2. **해결 방법**
   ```excel
   # 형식 통일: A2를 텍스트로 변환
   =VLOOKUP(TEXT(A2,"0"), $D$2:$E$100, 2, 0)

   # 공백 제거
   =VLOOKUP(TRIM(A2), $D$2:$E$100, 2, 0)

   # 둘 다 적용
   =VLOOKUP(TRIM(TEXT(A2,"0")), $D$2:$E$100, 2, 0)
   ```

3. **재발 방지 팁**
   - ERP에서 데이터 추출 시 숫자 코드가 텍스트로 저장되는 경우 많음
   - 매핑 전 `=ISNUMBER(A2)` 로 형식 확인 습관화

4. **관련 함수 추천**
   - `XLOOKUP`: VLOOKUP보다 유연하고 오류 처리 쉬움
   ```excel
   =XLOOKUP(TRIM(A2), $D$2:$D$100, $E$2:$E$100, "없음")
   ```

## ✅ 사용 예시 2 — ERP 처리 방식

**입력:**
```
[문제 유형] ERP 처리 방식

[상황]
SAP에서 전기된 전표를 수정하려고 하는데 직접 수정이 안 돼.
역분개 후 재전기해야 한다고 하는데 왜 직접 수정이 안 되는 건지,
그리고 역분개하는 방법도 알려줘.

[출력 형식]
1. 원인 분석
2. 해결 방법
3. 재발 방지 팁
4. 관련 기능 추천
```

---

**출력 예시:**

1. **원인 분석**
   SAP 등 ERP는 전기된 전표의 감사추적(Audit Trail) 보존을 위해 직접 수정 불가. 오류 수정은 반드시 역분개 후 재전기 방식으로 처리.
   → K-IFRS 제1008호 문단42: 전기오류 수정은 발견된 기간에 소급재작성이 원칙
     링크: https://kasb.or.kr (기준서 → K-IFRS 제1008호)

2. **해결 방법 (SAP 기준)**
   - FB08 (역분개 트랜잭션) 실행
   - 원전표 번호 입력 → 역분개 사유 코드 선택
   - 역분개 전표 생성 확인 후 → FB01로 정확한 내용 재전기

3. **재발 방지 팁**
   - 전기 전 미리보기(Simulate) 기능 활용 습관화
   - 전기 권한과 검토 권한 분리 → 오류 사전 방지 (권한분리 통제)

4. **관련 기능 추천**
   - SAP 전표 조회: FB03
   - 계정별 잔액 확인: FS10N
